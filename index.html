HTML

  • Give content structure via “elements”
  • Element and tag are often used interchangeably element usually refers to everything inside opening and closing tags, whereas tag refers only to what's inside angle brackets
  • Attributes define additional information about an element (always defined at opening of element)
  • HTML elements are case sensitive
  • head element = container for Metadata (info ~ title, accessibility, search engines, and performance but not seen by user)
  • Elements that are nested inside other elements (ie the parent element) are called child elements
  • h1-h6 should be done in order for accessibility reasons and h1 only used once

Git

  • git status: checks what branch you are currently on
  • git checkout -b branch-name: -b creates/names a new branch and checkout switches to it
  • git checkout branch-name: switches to existing branch
  • git add -A: adds all modifications made to working branch to staging area
  • git commit -m "description": commits all changes that have been added to staging area, along with message describing the changes
  • git pull origin branchname: receive a branches modifications to local environment
  • git push origin branchname: sends changes to specified remote branch (push can also create a new branch)
